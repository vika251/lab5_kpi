# Лабораторна робота №5: Автоматизація тестування веб-додатків з Selenium WebDriver

**Виконала:** студентка ІІІ курсу групи КП-33, Храполович Вікторія  
**Перевірив:** Олексій Артур

Цей репозиторій містить автоматизовані тести для веб-додатку [the-internet.herokuapp.com](https://the-internet.herokuapp.com/), виконані в рамках лабораторної роботи (Варіант 3).

## Опис
Проєкт реалізовано мовою **C#** з використанням фреймворку **NUnit** та бібліотеки **Selenium WebDriver**. 
Архітектура проєкту базується на використанні базового класу `BaseTest` для керування життєвим циклом браузера (Setup/Teardown).

## Технології
* **Мова:** C# (.NET 9.0)
* **Тестовий фреймворк:** NUnit
* **Інструмент автоматизації:** Selenium WebDriver
* **Браузер:** Google Chrome
* **Додаткові бібліотеки:** DotNetSeleniumExtras.WaitHelpers (для явних очікувань)

## Реалізовані тести (Варіант 3)

В проєкті автоматизовано 10 сценаріїв тестування згідно з завданням:

1.  **Checkboxes** (`/checkboxes`)
    * Перевірка початкового стану чекбоксів.
    * Перевірка зміни стану чекбокса після кліку.
2.  **Add/Remove Elements** (`/add_remove_elements/`)
    * Додавання нового елемента (кнопки "Delete").
    * Перевірка видалення елемента.
3.  **Dropdown** (`/dropdown`)
    * Вибір опції з випадаючого списку за текстом ("Option 1").
    * Перевірка, що опція успішно обрана.
4.  **Inputs** (`/inputs`)
    * Введення числових значень у поле вводу.
    * Перевірка правильності збереженого значення.
5.  **Status Codes** (`/status_codes`)
    * Перехід на сторінку статусу 200.
    * Перевірка URL та повідомлення про успішний статус код.
6.  **Nested Frames** (`/nested_frames`)
    * Робота з вкладеними фреймами (`frame-top` -> `frame-middle`).
    * Перевірка наявності тексту всередині конкретного фрейму.
7.  **Form Authentication** (`/login`)
    * Введення валідних даних для входу.
    * Перевірка успішної авторизації (Flash message) та наявності кнопки Logout.
8.  **Dynamic Content** (`/dynamic_content`)
    * Оновлення сторінки.
    * Перевірка зміни динамічного тексту після перезавантаження.
9.  **Disappearing Elements** (`/disappearing_elements`)
    * Перевірка видимості стабільних елементів меню (Home).
    * Перевірка кількості елементів у списку меню.
10. **Hover** (`/hovers`)
    * Емуляція наведення курсору миші на зображення користувача.
    * Використання `WebDriverWait` для очікування появи прихованого тексту.
